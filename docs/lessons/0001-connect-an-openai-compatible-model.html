接入一个真实模型

目标：在 10 分钟内把公众号写作 Agent 从 Mock 模式切换到 OpenAI-compatible 模型。

先理解一件事

应用最终会请求：AGENT_BASE_URL + /chat/completions。请求头使用 Authorization: Bearer API_KEY，请求正文中携带 AGENT_MODEL。所以你只需要从供应商那里找到三个值：根地址、密钥、模型 ID。

步骤 1：准备供应商信息

打开供应商的官方 API 文档和控制台，找到 Chat Completions 兼容地址、创建一个 API Key，并复制准确的模型 ID。如果文档只支持别的接口形状，当前应用不能直接接入。

步骤 2：创建本机配置
cd C:\Users\lhs04\Documents\小程序\公众号agent
Copy-Item .env.example .env.local
notepad .env.local

把文件改成：

AGENT_PROVIDER_MODE=openai-compatible
AGENT_BASE_URL=https://你的服务地址/v1
AGENT_API_KEY=你的本机密钥
AGENT_MODEL=供应商提供的模型ID
AGENT_TIMEOUT_MS=45000
不要把 API Key 发给我，也不要写进 public/app.js。`.env.local` 已被项目忽略，只由本机 Node 服务读取。
步骤 3：启动应用
node backend/server.mjs

若没有全局 Node：

& 'C:\Users\lhs04\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' backend/server.mjs
步骤 4：先检查模式
Invoke-RestMethod http://127.0.0.1:3000/api/health

期望看到 providerMode 为 openai-compatible。这只证明配置被读取，还没有产生模型费用。

步骤 5：完成一次真实请求

打开 http://127.0.0.1:3000，新建一个项目并点击“开始梳理”。如果出现模型生成的第一条追问，接入完成。

检索练习：不看上文，回答：为什么 Base URL 里不能再写一次 /chat/completions？如果你能回答“因为应用代码会自动附加它”，就掌握了最常见的路径错误。

排错顺序

  1. 健康检查仍显示 mock：确认文件名必须是 .env.local，然后重启 Node。
  2. 401：检查 Key 是否有效、是否属于当前供应商。
  3. 404：检查 Base URL 是否重复包含接口路径。
  4. 429：检查余额、并发和速率额度。
  5. 结构校验失败：换用更擅长遵循 JSON 指令的模型。

主要参考：OpenAI Chat Completions 官方接口；随手查看 模型接入速查表。
